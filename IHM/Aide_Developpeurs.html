Error messages when linking TRUST

ld32: ERROR   33 : Unresolved text symbol "Mod_turb_hyd_ss_maille_VEF::mettre_a_jour(double)" -- 1st referenced by TRUST_opt.a(Turb_hyd_SGE_VEF.o).

Reason:
The method is defined in the .h file but not coded either in .h (if inlined) or .cpp file.

Correction:
Suppress the obsolete definition in the .h file or code the method in .h (if inlined) or in .cpp file.

make Modturbhybase.o
Modturbhybase.cpp -DNO_PVM_ -DSGI -D_INLINE_EVALUATEURS_ -DNDEBUG
"Modturbhybase.cpp", line 377: error(1289): the object has type qualifiers
          that are not compatible with the member function
       loipar.imprimer_ustar(os);
       ^

1 error detected in the compilation of "Modturbhybase.cpp".
*** Error code 2 (bu21)

Raison:
-------
La methode imprimer_ustar de type non const peut modifier
l'objet loipar or cet appel est dans une methode const.

Correction:
-----------
Rendre const la methode imprimer_ustar


declare et implemente

declare_base(classe) dans le .h
implemente_base(classe,"classe",classe_mere) dans le .cpp

L'objet ne pourra etre instancie contrairement a une classe
avec :
declare_instanciable(classe) dans le .h
implemente_instanciable(classe,"classe",classe_mere) dans le .cpp
Le mot cle utilise est defini entre les ""

Attention: On peut avoir Segmentation Fault lorsque par megarde on utilise
le mot cle alors que la classe n'est pas instanciable.
